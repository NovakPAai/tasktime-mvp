# Skill: ru-compliance-context

Контекст для юридических/регуляторных решений в РФ. Не заменяет юристов и ИБ‑отдел.

1. TaskTime **не относится к КИИ** по умолчанию. Нельзя автоматически навешивать на него самые жёсткие требования КИИ/ГОСТ без отдельного решения заказчика.
2. Персональные данные (ФИО, e‑mail, логины) должны обрабатываться в соответствии с ФЗ‑152: хранение и обработка данных в РФ, защищённые каналы связи, ограничение доступа по ролям.
3. Все решения по хранению логов, аудитам и резервному копированию нужно принимать с учётом политики конкретного заказчика (банк/финорганизация), а не абстрактных «лучших практик».
4. Любой спорный вопрос (хранить ли X, логировать ли Y, где размещать Z) — эскалировать человеку (PO/ИБ/юрист), **а не принимать автоматически** в коде или конфигурации.
5. При проектировании интеграций (GitLab, Telegram, Confluence) учитывать требования заказчика по размещению сервисов (on‑prem/облако в РФ) и передаче данных за рубеж — такие решения тоже не принимать автоматически.
