# Что было сделано при запуске TaskTime (для отчёта / объяснения)

Краткое описание шагов, чтобы можно было понятно рассказать, что сделано, в том числе людям без технического бэкграунда.

---

## 1. Подготовка сервера

- **Где:** виртуальная машина (VPS) в облаке Timeweb, операционная система Ubuntu.
- **Что сделано:** на сервер установлены и настроены:
  - **Node.js** — среда для работы приложения;
  - **PostgreSQL** — база данных (задачи, пользователи, учёт времени);
  - **Nginx** — веб-сервер, который принимает запросы из интернета и передаёт их приложению;
  - **системный сервис** — чтобы приложение автоматически запускалось и перезапускалось при сбоях;
  - **файрвол (UFW)** — ограничен доступ к серверу только по нужным портам (сайт, SSH);
  - **Certbot** — для возможности в будущем включить HTTPS (защищённое соединение).

Итог: сервер готов к размещению приложения и базе данных.

---

## 2. База данных

- **Что сделано:**
  - Создана база данных с именем `tasktime`.
  - Создан отдельный пользователь БД с паролем (доступ только к этой базе).
  - Настроены правила доступа к БД (подключение только с самого сервера по паролю).
  - В базе созданы таблицы: пользователи, задачи, учёт времени (по схеме проекта).

Итог: база готова хранить пользователей, задачи и записи времени.

---

## 3. Размещение кода приложения

- **Что сделано:**
  - Код проекта (репозиторий GitHub: jackrescuer-gif/tasktime-mvp) скопирован на сервер в каталог приложения.
  - Установлены зависимости приложения (npm install).
  - Настроен конфиг: пароль БД, секретный ключ для авторизации, порт — всё записано в файл окружения (.env), приложение его читает при старте.
  - Настроен запуск приложения как системной службы: приложение работает в фоне и поднимается после перезагрузки сервера.

Итог: приложение установлено на сервере и запущено.

---

## 4. Прокси и доступ из интернета

- **Что сделано:**
  - Nginx настроен как «прокси»: запросы по адресу сервера (по 80 порту) перенаправляются на наше приложение (порт 3000 внутри сервера).
  - Открыт доступ по HTTP к приложению с любого IP.

Итог: приложение доступно в браузере по адресу сервера (например, http://5.129.242.171).

---

## 5. Учётные записи пользователей

- **Что сделано:**
  - Созданы демо-пользователи для проверки (alice@demo.com и др., пароль demo123).
  - Созданы учётные записи команды: Павел, Георгий, Олеся, Андрей, Антон (логины в латинице, формат и пароли описаны в файле ACCOUNTS.md).

Итог: можно входить в систему под своими учётками и тестовыми.

---

## 6. Деплой обновлений в будущем

- На сервере подготовлен скрипт обновления: подтягивание нового кода из GitHub, переустановка зависимостей и перезапуск приложения одной командой (или через панель/CI при желании).
