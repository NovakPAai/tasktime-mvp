# Design System Source of Truth (SSOT)

Этот проект следует строгому правилу: **НИКАКОГО ХАРДКОДА ЦВЕТОВ И ОТСТУПОВ.**

## 1. Источник правды
Единственным местом, где разрешены HEX-коды и PX-значения, является файл:
`frontend/src/design-tokens.ts`

## 2. Правила для ИИ и разработчиков
- **ЗАПРЕЩЕНО** использовать HEX-коды (напр. `#FFFFFF`) в `.css`, `.tsx` или `.ts` файлах (кроме самого файла токенов).
- **ЗАПРЕЩЕНО** использовать произвольные PX-значения для отступов и скруглений.
- **ОБЯЗАТЕЛЬНО** использовать только CSS-переменные `var(--...)` в CSS-файлах.
- **ОБЯЗАТЕЛЬНО** использовать объект `tokens` из `design-tokens.ts` в React-компонентах.

## 3. Маппинг
Все токены из `design-tokens.ts` автоматически зеркалируются в `tokens.css` в формате:
`camelCase` (JS) -> `--kebab-case` (CSS)

Пример:
- JS: `tokens.dark.bgMain`
- CSS: `var(--bg-main)`

## 4. Как вносить изменения
Если нужно изменить цвет или отступ:
1. Отредактируйте `frontend/src/design-tokens.ts`.
2. Запустите `npm run tokens:sync` (автоматически обновит `tokens.css`).
3. Убедитесь, что изменения подтянулись в `App.tsx` через `ConfigProvider`.

**Любое отклонение от этой системы считается багом.**
