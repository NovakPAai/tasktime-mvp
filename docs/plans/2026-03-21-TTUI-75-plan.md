# [TTUI-75] Implementation Plan — Прогресс дочерних задач и трудозатраты

**Дата:** 2026-03-21
**Ветка:** `UxUi/clever-chaum`
**ТЗ:** `docs/tz/TTUI-75.md`

## Goal
Добавить в список задач: прогресс-бар дочерних, `estimatedHours`, совокупную оценку по иерархии.

## Architecture

```
Backend (issues.service.ts)
  listIssues() → добавить estimatedHours + childrenStats + aggregatedEstimatedHours
  ↓
Frontend
  IssueProgressBar (новый компонент)
  ProjectDetailPage — колонка estimatedHours + прогресс
  IssueDetailPage — прогресс + совокупная оценка
  IssuePreviewDrawer — прогресс + совокупная оценка
```

## Definition of Done (из ТЗ)
- [ ] Прогресс-бар только у задач с дочерними
- [ ] Логика %/дробь работает корректно
- [ ] estimatedHours в списке (N.Nh)
- [ ] Совокупная оценка на родительских задачах
- [ ] Прогресс в IssueDetailPage и IssuePreviewDrawer
- [ ] Нет N+1 запросов
- [ ] Тесты зелёные, lint OK

## Tasks

### Task 1 — Backend: childrenStats в API (TTUI-76)
1. Читаем `issues.service.ts` → `listIssues()`
2. Добавляем рекурсивный include дочерних (до 4 уровней) с `estimatedHours` и `status`
3. Добавляем функцию `calculateChildrenStats(children)` → `{total, done, allEstimated, totalEstimatedHours, doneEstimatedHours, progressPercent}`
4. Добавляем `estimatedHours` в select списка
5. Тест (unit): `calculateChildrenStats` — edge cases
6. Тест (integration): `GET /projects/:id/issues` → поля в ответе

### Task 2 — Backend: aggregatedEstimatedHours (TTUI-77)
1. Добавляем функцию `sumEstimatedHours(issue)` — рекурсивно суммирует own + все дочерние
2. Добавляем поле `aggregatedEstimatedHours` в маппинг ответа
3. Тест (unit): `sumEstimatedHours`

### Task 3 — Frontend: компонент IssueProgressBar (TTUI-78)
1. Создаём `frontend/src/components/issues/IssueProgressBar.tsx`
2. Props: `childrenStats`, `size?`
3. allEstimated=true → `<Progress percent>`, иначе → `done/total`
4. Тест: рендер обоих режимов

### Task 4 — Frontend: estimatedHours в таблице (TTUI-79)
1. Расширяем тип `Issue` в `types/index.ts` — добавляем `childrenStats`, `aggregatedEstimatedHours`
2. В `ProjectDetailPage.tsx` → добавляем колонку между Priority и Assignee
3. Формат: `4.5h` (серый), `—` если null
4. Прогресс-бар в колонке Title для задач с дочерними

### Task 5 — Frontend: совокупная оценка (TTUI-80)
1. В колонке estimatedHours для родительских задач (`_count.children > 0`) показываем `Σ 24.5h`
2. Tooltip: «Суммарно по всей иерархии»

### Task 6 — Frontend: интеграция в DetailPage + Drawer (TTUI-81)
1. `IssueDetailPage.tsx` → добавляем `<IssueProgressBar>` и `aggregatedEstimatedHours` в сайдбар
2. `IssuePreviewDrawer.tsx` → то же самое
