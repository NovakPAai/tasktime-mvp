# Массовые операции (TTBULK-1)

Инструмент для массового изменения задач: смена статуса, исполнителя, полей, комментариев, удаления и т.д.

## Доступ

Операции доступны пользователям с системной ролью **BULK_OPERATOR**. Администратор выдаёт её:
- **Напрямую:** Админ → Пользователи → выбрать юзера → «Системные роли» → добавить BULK_OPERATOR.
- **Через группу:** Админ → Группы пользователей → открыть группу → вкладка «Системные роли» → «Выдать системную роль» → BULK_OPERATOR.

Роль наследуется всеми членами группы; инвалидация кэша эффективных ролей — до 60 секунд.

## Запуск операции из UI

1. Откройте **Поиск задач** (`/search`) и введите JQL или выберите сохранённый фильтр.
2. Выделите задачи чекбоксами в таблице.
3. Нажмите **«Массовые операции»** (⚡️ кнопка справа).
4. Wizard проведёт через 4 шага:
   - **Шаг 1 — Операция:** выберите тип (перевод статуса, назначение исполнителя, редактирование поля, добавление комментария, перемещение в спринт, удаление).
   - **Шаг 2 — Настройка:** зависит от типа (UUID-ы, текст, даты — ID-first UX).
   - **Шаг 3 — Предпросмотр:** сервер резолвит scope и покажет **eligible / skipped / conflicts** счётчики. Virtualized-списки для длинных выборок.
   - **Шаг 4 — Подтверждение:** summary payload'а + для DELETE — ввести «DELETE» (заглавными).
5. Нажмите **«Запустить»**.

После submit'а:
- Справа внизу появится **chip** с прогрессом (live через SSE).
- Drawer откроется автоматически — видны **Выполнено / Ошибок / Пропущено / Обработано** счётчики, ETA.
- Когда операция завершится — status обновится (SUCCEEDED / PARTIAL / FAILED / CANCELLED).

## Управление активными операциями

- **Отменить:** в drawer'е кнопка «Отменить» (Popconfirm). Processor завершит активный batch, остальные items помечает CANCELLED_BY_USER.
- **Скачать отчёт CSV:** после завершения (SUCCEEDED / PARTIAL) — кнопка «Скачать отчёт CSV».
- **Повторить failed items (Retry):** на странице `/operations/:id` если есть failed — кнопка «Retry failed». Новая операция с scope=ids из оригинала.

## Страница /operations

Доступна через sidebar («Массовые операции»). Показывает список ваших операций за 30 дней:
- Фильтры по статусу и типу.
- Retry / Открыть детальную страницу.
- Админский фильтр «Все операции» — **в roadmap** (требует отдельный backend endpoint).

## История изменений задачи

На странице задачи (`/issues/:id`) в секции **История изменений**: записи, пришедшие из массовой операции, помечены бейджем **«Массовая операция»** — клик → детальная страница операции.

## Ограничения

- **maxConcurrentPerUser = 3** (по умолчанию) — не больше 3 активных операций на пользователя одновременно. Super Admin настраивает в `/admin/system` → «Массовые операции» (от 1 до 20).
- **maxItems = 10000** — hard-cap DTO; runtime ceiling настраивается Super Admin'ом (от 100 до 10000).
- **Preview TTL = 15 минут** — если долго не подтверждать submit, preview token истечёт (409 PREVIEW_EXPIRED → автоматический re-preview в wizard).
- **DELETE** требует фразы `DELETE` (заглавными) на шаге 4.

## Форензика

Изменения отражаются в `AuditLog` с `bulkOperationId` (FK на BulkOperation.id). SQL-запросы для SRE — см. [OPERATIONS_RUNBOOK.md § Bulk Operations](../OPERATIONS_RUNBOOK.md#bulk-operations).

## Rollback

Flip backend: `FEATURES_BULK_OPS=false` в env + рестарт → `/api/bulk-operations/*` → 404. Frontend: `VITE_FEATURES_BULK_OPS=false` + rebuild → wizard кнопка скрыта, sidebar-link скрыт.
