# AI-ассистент — Руководство пользователя

> Функция: Sprint 4 (Claude API integration)
> Доступ: MANAGER, ADMIN (управление); все (просмотр)
> Обновлено: 2026-03-25

---

## Что умеет AI-ассистент?

Flow Universe интегрирован с Claude AI (Anthropic). Ассистент может:

1. **Оценить трудоёмкость** задачи — предложить количество часов на основе описания
2. **Декомпозировать** задачу — разбить на подзадачи автоматически

---

## AI-флаги на задаче

Каждая задача имеет три AI-поля (видимы и редактируемы для MANAGER, ADMIN):

| Поле | Значения | Описание |
|------|---------|---------|
| **AI-eligible** | Да / Нет | Разрешено ли AI работать с этой задачей |
| **Тип исполнителя** | HUMAN / AGENT / MIXED | Кто будет выполнять задачу |
| **AI-статус** | NOT_STARTED / IN_PROGRESS / DONE / FAILED | Статус выполнения агентом |

---

## Оценка трудоёмкости (Estimate)

1. Откройте задачу с заполненным описанием
2. Убедитесь, что **AI-eligible** = Да
3. Нажмите кнопку **AI: Оценить** (в разделе AI на карточке задачи)
4. AI вернёт:
   - Рекомендуемое количество часов
   - Обоснование оценки
5. Нажмите **Принять** — оценка запишется в поле «Оценка (часы)»

---

## Декомпозиция (Decompose)

1. Откройте задачу типа EPIC, STORY или TASK
2. Нажмите кнопку **AI: Декомпозировать**
3. AI предложит список подзадач с оценками
4. Выберите подзадачи для создания (чеклист)
5. Нажмите **Создать подзадачи** — они появятся в иерархии задачи

---

## Журнал AI-сессий

На карточке задачи → вкладка **AI-сессии** (если были запуски):

| Поле | Описание |
|------|---------|
| Модель | Версия Claude, использованная для задачи |
| Токены | Входящие / исходящие |
| Стоимость | В USD |
| Дата | Когда запускалась сессия |

---

## Записи времени от AI

Когда AI-агент выполняет задачу, время записывается автоматически в **time_logs** с меткой источника `AGENT`. Стоимость API-вызова также фиксируется.

Это позволяет сравнивать стоимость человека vs. агента по задаче.

---

## Требования для включения

Для работы AI-ассистента необходимо:
- `ANTHROPIC_API_KEY` — установлен в переменных среды (администратор)
- Feature flag `ai` — включён (проверьте у администратора)
- Задача имеет достаточно подробное описание для анализа
