# Отчёты — Руководство пользователя

> Раздел: Администрирование → Отчёты
> Доступ: VIEWER, MANAGER, ADMIN
> Обновлено: 2026-03-25

---

## Где найти отчёты?

Меню слева → **Администрирование** → вкладка **Отчёты**.

---

## Отчёт: Задачи по статусам

**Показывает:** сколько задач в каждом статусе (OPEN, IN_PROGRESS, REVIEW, DONE, CANCELLED).

**Настройки фильтра:**
| Поле | Обязательно | Описание |
|------|-------------|---------|
| **Проект** | ✓ | Выберите проект |
| **Спринт** | — | Ограничить конкретным спринтом |
| **Период (от)** | — | Дата создания задачи — начало |
| **Период (до)** | — | Дата создания задачи — конец |

**Пример результата:**
```
OPEN         14 задачи
IN_PROGRESS   6 задач
REVIEW        3 задачи
DONE         32 задачи
CANCELLED     2 задачи
```

**Как использовать:** Контролируйте накопление задач в колонке OPEN — если их много и мало IN_PROGRESS, команда возможно перегружена или задачи не декомпозированы.

---

## Отчёт: Задачи по исполнителям

**Показывает:** сколько задач у каждого участника команды.

**Те же фильтры**, что и в отчёте по статусам.

**Пример результата:**
```
Иванов И.    8 задач
Петрова А.   5 задач
Сидоров В.   3 задачи
Без исполнителя  6 задач
```

**Как использовать:** Проверяйте равномерность нагрузки. Если у одного разработчика 15 задач, а у другого 2 — перераспределите.

## Системная статистика

Вкладка **Дашборд** в разделе Администрирования:

| Метрика | Описание |
|---------|---------|
| Аптайм | Время работы без перезапуска |
| RAM | Используемая память |
| DB | Задержка базы данных |
| Redis | Статус кэша |
| Ошибки за 24ч | Количество серверных ошибок |

---

## Журнал аудита

Вкладка **Активность** (для ADMIN, MANAGER, VIEWER):
- Кто, когда, что изменил
- IP-адрес
- Браузер/клиент
- Тип действия (issue.created, issue.status_changed, и т.д.)
- Пагинация (50 записей на страницу)

Используется для контроля доступа (ФЗ-152).

---

## Экспорт данных

В текущей версии MVP экспорт данных доступен через API:

```bash
# Задачи проекта TTMP
GET /api/projects/{id}/issues

# Время по проекту
GET /api/time-logs?from=2026-01-01&to=2026-03-31
```

В следующих версиях планируется встроенный экспорт в CSV/Excel.
