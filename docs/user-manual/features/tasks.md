# Задачи — Руководство пользователя

> Раздел: Управление задачами (Issues)
> Обновлено: 2026-03-25

---

## Типы задач

Flow Universe использует иерархию из 5 типов:

| Тип | Иконка | Может быть родителем | Может быть дочерним |
|-----|--------|---------------------|---------------------|
| **Эпик (EPIC)** | 🟣 | История, Задача | — (верхний уровень) |
| **История (STORY)** | 🟢 | Задача, Подзадача | Эпик |
| **Задача (TASK)** | 🔵 | Подзадача | Эпик, История |
| **Подзадача (SUBTASK)** | ⬜ | — (нижний уровень) | История, Задача |
| **Баг (BUG)** | 🔴 | Подзадача | Эпик, История |

---

## Статусы задачи

```
OPEN → IN_PROGRESS → REVIEW → DONE
                   ↘ CANCELLED
```

| Статус | Описание |
|--------|---------|
| **Открыта (OPEN)** | Задача создана, работа не начата |
| **В работе (IN_PROGRESS)** | Активная разработка |
| **На ревью (REVIEW)** | Ожидает проверки |
| **Готово (DONE)** | Завершена |
| **Отменена (CANCELLED)** | Закрыта без выполнения |

---

## Приоритеты

| Приоритет | Когда использовать |
|-----------|-------------------|
| **Критический (CRITICAL)** | Блокирует работу системы или всей команды |
| **Высокий (HIGH)** | Важная функция или серьёзный баг |
| **Средний (MEDIUM)** | Стандартные задачи (по умолчанию) |
| **Низкий (LOW)** | Улучшения, задачи «когда будет время» |

---

## Ключ задачи

Каждая задача получает уникальный ключ: `КОД_ПРОЕКТА-НОМЕР`

Примеры: `TTMP-42`, `DEMO-7`, `BACK-103`

Ключ используется для ссылок, поиска и интеграции с GitLab.

## Создание задачи

1. Откройте **Проекты** → выберите проект
2. Нажмите кнопку **+ Задача** (или **+ Issue** в шапке списка)
3. Заполните форму:

| Поле | Обязательно | Описание |
|------|-------------|---------|
| **Название** | ✓ | Краткое описание задачи |
| **Тип** | — | По умолчанию: Задача |
| **Приоритет** | — | По умолчанию: Средний |
| **Исполнитель** | — | Назначить сразу (MANAGER, ADMIN) |
| **Спринт** | — | Добавить в спринт сразу |
| **Описание** | — | Подробности, шаги воспроизведения (для Бага) |
| **Оценка (часы)** | — | Плановые трудозатраты |
| **Родительская задача** | — | Для иерархии |

4. Нажмите **Создать**

## Карточка задачи

Откройте задачу кликом по названию. Доступные действия:

### Верхняя панель
- **Ключ задачи** (TTMP-42) — отображается в заголовке
- **Редактировать** — изменить название, описание, поля
- **Удалить** — только ADMIN

### Основные поля (правая колонка)
| Поле | Кто может менять |
|------|-----------------|
| Статус | Все |
| Приоритет | MANAGER, ADMIN |
| Тип | MANAGER, ADMIN |
| Исполнитель | MANAGER, ADMIN |
| Спринт | MANAGER, ADMIN |
| Релиз | MANAGER, ADMIN |
| Оценка (часы) | MANAGER, ADMIN |

### Вкладки внизу
- **Комментарии** — обсуждение задачи
- **Время** — записи о затраченном времени + кнопки Старт/Стоп
- **История** — лог всех изменений (кто, когда, что изменил)
- **Подзадачи** — дочерние задачи

---

## Редактирование задачи

Клик по любому полю на карточке задачи переводит его в режим редактирования. После изменения нажмите Enter или кликните вне поля для сохранения.

Название и описание редактируются через форму редактирования (кнопка «Изменить»).

---

## Изменение статуса

Три способа:
1. **На карточке задачи** — выпадающий список статусов
2. **На Kanban-доске** — перетащить карточку в другую колонку
3. **В списке задач** — клик по статусу в таблице

---

## Иерархия задач

Чтобы добавить подзадачу:
1. Откройте родительскую задачу
2. На вкладке **Подзадачи** нажмите **+ Добавить**
3. Выберите тип (Подзадача или Баг) и заполните название

Или при создании задачи укажите поле **Родительская задача**.

---

## Поиск и фильтрация

В списке задач проекта доступны фильтры:

| Фильтр | Значения |
|--------|---------|
| Статус | OPEN, IN_PROGRESS, REVIEW, DONE, CANCELLED |
| Тип | EPIC, STORY, TASK, SUBTASK, BUG |
| Приоритет | CRITICAL, HIGH, MEDIUM, LOW |
| Исполнитель | Выбор из списка пользователей |
| Спринт | Список спринтов проекта |
| Поиск | Полнотекстовый поиск по названию |
| Период | Дата создания: от / до |

Несколько значений в одном фильтре работают как OR (например: статус = OPEN, IN_PROGRESS).

## AI-флаги (для MANAGER, ADMIN)

На карточке задачи есть раздел **AI-настройки**:

| Поле | Описание |
|------|---------|
| **AI-eligible** | Может ли задача быть выполнена AI-агентом |
| **Тип исполнителя** | HUMAN / AGENT / MIXED |
| **Статус AI** | NOT_STARTED → IN_PROGRESS → DONE / FAILED |

Подробнее — в [AI-ассистент](./ai-assistant.md).
