# Поиск задач — Руководство пользователя

> Обновлено: 2026-04-21 (TTSRH-1 PR-21)
> URL: `/search`, feature flag `FEATURES_ADVANCED_SEARCH`
> Язык запросов: [jql.md](./jql.md)

Страница **Поиск задач** (Главное меню → Поиск задач) — единое место для всех поисков по задачам: от «мои срочные» до сложных межпроектных отчётов. Работает в двух режимах: **Basic** (визуальный конструктор) и **Advanced** (текстовое поле TTS-QL).

## Basic vs Advanced

### Basic (визуальный конструктор)

Каждое условие — **чип** (chip). Клик по чипу → popover с автокомплитом значений. Кнопка `+` → каскадное меню полей, разбитое на категории (Задача / Даты / Пользователи / Планирование / AI / Кастомные).

**Когда использовать:** простые фильтры из 2-5 условий, соединённых через `AND`.

**Ограничения:** не поддерживает `OR`, `NOT`, скобки. Если вручную набрать в Advanced что-то с `OR`, кнопка переключения в Basic будет **заблокирована** с подсказкой почему.

### Advanced (TTS-QL)

Полнотекстовое поле с **подсветкой синтаксиса** и **автокомплитом** (Ctrl+Space или автоматически при наборе):

- после `AND` / `OR` / `NOT` / `(` → предлагает **поля**;
- после поля → **операторы**, совместимые с типом поля;
- после оператора → **значения** (аватары пользователей, цветные точки статусов, имена релизов);
- после `ORDER BY` → только **sortable-поля**.

**Inline-ошибки:** подчёркивание токена с ошибкой + красный блок внизу с кодом и подсказкой (до 5 сообщений; больше — показывает счётчик).

**Shortcuts:**

| Клавиша | Действие |
|---------|----------|
| `/` | фокус в редактор |
| `Esc` | сброс фокуса |
| `Ctrl+Enter` | выполнить запрос |
| `Ctrl+S` | сохранить фильтр |
| `Ctrl+Shift+S` | сохранить как… |

## Сохранённые фильтры

Слева колонка с категориями:

- **Избранные** — ваши отмеченные ★ фильтры.
- **Общие** — публичные (`PUBLIC`) фильтры всех пользователей.
- **Поделены с вами** — `SHARED` фильтры, в которые вас добавили.
- **Мои** — ваши `PRIVATE` / `SHARED` / `PUBLIC` фильтры.
- **Недавние** — использованные за последние 7 дней.

### Создание

Ввести JQL → кнопка **«+ Сохранить»** в sidebar (или `Ctrl+S`) → открывается модалка:

| Поле | Значение |
|------|----------|
| Имя | обязательно |
| Описание | опционально |
| Видимость | `PRIVATE` (по умолчанию) / `SHARED` / `PUBLIC` |
| При `SHARED` | мультиселект пользователей + групп + permission (`READ` / `WRITE`) |
| Избранный | чекбокс |

### Шаринг

На любом собственном фильтре → кнопка «Поделиться» → модалка аналогичная созданию, позволяет изменить `visibility`, список пользователей и permission. Также даёт прямую **ссылку** на фильтр (`/search/saved/:id`) — по ней можно открыть в новой вкладке / скопировать в мессенджер.

### Права

| Уровень | Кто видит | Кто редактирует |
|---------|-----------|------------------|
| `PRIVATE` | только owner | только owner |
| `SHARED` | owner + entries в `SavedFilterShare` | owner, а также те shared-users, у кого permission = `WRITE` |
| `PUBLIC` | все аутентифицированные | только owner |

**Важно про `PUBLIC`:** выполнение PUBLIC-фильтра всё равно использует accessible-projects **читающего** — никакой пользователь не получит задачи вне своих прав через чужой фильтр (R3 scope).

## Колонки

Дефолтный набор: `key, summary, type, status, priority, assignee, sprint, updated`.

### Конфигуратор

Клик на **⚙ Колонки** в панели результатов → две колонки:

```
Доступные                 Выбранные
─────────                 ──────────
 creator                   key
 project                   summary
 projectKey               →type
 release       ← → →       status
 due                       priority
 description               assignee
 …                         sprint
                           updated
```

- Левая/правая стрелки — добавить/убрать.
- Drag-and-drop в правой колонке — порядок.
- Сортировка колонки — клик по заголовку (переписывает `ORDER BY` в JQL).

### Сохранение

- **В рамках текущего фильтра** — автоматически в `SavedFilter.columns` при save.
- **Как дефолт для меня** — кнопка «Сохранить как дефолт» → `PATCH /users/me/preferences`.

## Экспорт

**⬇ Экспорт** → CSV или XLSX. Экспорт стримовый — работает для любого размера результата (лимит `/search/issues` не применяется).

- **CSV** — UTF-8 с BOM, разделитель `;` (для Excel-ru), первая строка — заголовки.
- **XLSX** — одна таблица, заголовки в первой строке, авто-ширина колонок.

## URL и шаринг ссылки

Текущее состояние страницы **отражено в URL**:

```
/search?jql=<url-encoded>&view=table&columns=key,status,assignee&page=2
```

- **Копировать URL** = копировать фильтр. Открывается у коллеги — увидит тот же результат (с учётом его scope-прав).
- **Reload** → восстанавливает всё: JQL, колонки, страница.
- **Back / Forward** в браузере → история фильтров (как в поисковике).

## Горячие сценарии

### Ежедневный standup

Сохранённый фильтр «Мои HIGH» + избранный ★:

```
assignee = currentUser() AND priority IN (CRITICAL, HIGH)
AND statusCategory != DONE
ORDER BY priority DESC, updated DESC
```

### Планирование спринта

```
project = TTMP AND sprint IS EMPTY AND statusCategory = TODO
AND priority IN (CRITICAL, HIGH) AND estimatedHours IS NOT EMPTY
ORDER BY priority DESC
```

### Release review

```
release = "TTMP-5.0" AND statusCategory != DONE
ORDER BY priority DESC, assignee ASC
```

### Checkpoint pre-flight

```
violatedCheckpointsOf("TTMP-5.0")
  OR (release = "TTMP-5.0" AND checkpointsAtRisk())
```

## Ограничения

- **Лимит на страницу:** 100 задач. Для больших выборок — CSV/XLSX export.
- **Rate-limit:** 30 запросов / минута / пользователь.
- **Timeout:** 10 сек на выполнение. Длительные запросы — сокращайте предикаты, добавляйте `project`-фильтр (сужает scope).
- **`WAS` / `CHANGED`** — синтаксис зарезервирован, но семантика в **Phase 2**.
- **Полнотекстовый поиск (`pg_trgm`)** — Phase 2.
