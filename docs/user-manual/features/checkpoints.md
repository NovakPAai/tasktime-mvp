# Контрольные точки релиза — Руководство пользователя

> Обновлено: 2026-04-19 (TTMP-160)

---

## Что такое контрольные точки?

**Контрольная точка (КТ)** — это заранее определённая дата, к которой задачи в релизе должны соответствовать заданным критериям (например, «все баги переведены в DONE за 3 дня до релиза» или «все эпики имеют назначенного исполнителя»). Нарушения агрегируются в **уровень риска** релиза: `LOW` / `MEDIUM` / `HIGH` / `CRITICAL`.

Модуль помогает релиз-менеджеру:
- видеть заранее, какие задачи не успевают попасть в релиз;
- применять к релизу готовый набор КТ одним действием (шаблон);
- получать агрегированную сводку риска и детализацию по каждой КТ;
- экспортировать нарушения в CSV для внешних отчётов.

---

## Где искать

- **Карточка релиза (глобальная страница «Релизы»)** — вкладка «Контрольные точки». Два вида:
  - **Список** (по умолчанию) — одна КТ = одна карточка со «светофором» и списком нарушений.
  - **Матрица** — строки = задачи, столбцы = КТ, цветные иконки состояния (🟢 passed / 🔴 violated / 🟡 pending / — n/a). Экспорт в CSV.
- **Карточка релиза** — вкладка «Диаграмма сгорания» — см. [release-burndown.md](./release-burndown.md).
- **Задача** (`IssueDetailPage`) — секция «Контрольные точки» показывает все КТ активных релизов, в которые входит задача.
- **Доска** (`BoardPage`) — индикатор ⚠️ на карточке задачи, если у неё хотя бы одна открытая violation.
- **Дашборд / TopBar** — бейдж «Мои нарушения» со счётчиком; по клику — фильтрованный список.

## Как работать (по ролям)

### Релиз-менеджер (`RELEASE_MANAGER` или проектный `MANAGER`)

1. **Применить шаблон к релизу**:
   - Откройте релиз → вкладка «Контрольные точки» → кнопка «Применить шаблон».
   - В модальном окне выберите шаблон из списка → предпросмотр покажет, какие задачи пройдут / нарушат каждую КТ.
   - Подтвердите — КТ создадутся на релизе с вычисленными дедлайнами (`plannedDate + offsetDays`).
2. **Добавить КТ вручную**: выберите один или несколько типов из выпадающего списка.
3. **Ручной пересчёт**: кнопка «Пересчитать» — запускает немедленный пересчёт (обычно не нужен; cron делает это каждые 10 мин).
4. **Удалить КТ**: иконка корзины на карточке КТ.

### Разработчик (`USER`)

- На своих задачах видит, какие КТ вы нарушаете и почему. Стандартный workflow (перевод в `DONE`, заполнение полей) автоматически снимает violations.
- В TopBar — бейдж «Мои нарушения» (FR-12); клик открывает список задач с нарушениями, сгруппированный по релизу.

### PM / продуктовый менеджер

- Доступ к вкладкам «Контрольные точки» и «Диаграмма сгорания» на любом релизе своих проектов.
- Применять шаблоны может, если имеет permission `RELEASES_EDIT` или системную роль `RELEASE_MANAGER`.

### Аудитор (`AUDITOR`)

- Read-only доступ ко всем релизам и КТ.
- `/admin/checkpoint-audit` — страница аудита: все события нарушений (открытие / закрытие), фильтры по периоду / проекту / типу / релизу, экспорт в CSV (UTF-8 BOM + CRLF для Excel).

## Типы и шаблоны (для `SUPER_ADMIN` / `ADMIN` / `RELEASE_MANAGER`)

- `/admin/release-checkpoint-types` — CRUD типов: название, цвет, вес (`CRITICAL` / `HIGH` / `MEDIUM` / `LOW`), смещение в днях от `plannedDate`, критерии (`STATUS_IN`, `DUE_BEFORE`, `ASSIGNEE_SET`, `CUSTOM_FIELD_VALUE`, `ALL_SUBTASKS_DONE`, `NO_BLOCKING_LINKS`).
- `/admin/release-checkpoint-templates` — CRUD шаблонов (именованный набор типов). Клон, bulk-apply на несколько релизов.
- **Sync-instances** — если у типа изменились критерии или offset, админ может синхронизировать уже созданные КТ на релизах одной операцией (FR-15).

## Матрица «Задачи × КТ»

Вкладка «Контрольные точки» → переключатель «Список / Матрица»:
- **строки** — задачи релиза;
- **столбцы** — КТ;
- **ячейки**:
  - 🟢 **passed** — задача прошла критерии КТ;
  - 🔴 **violated** — задача нарушила КТ (tooltip с причиной);
  - 🟡 **pending** — дедлайн КТ ещё не наступил;
  - `—` **n/a** — задача не попадает под `issueTypes`-фильтр критерия (например, KT применима только к `BUG`, а задача — `TASK`).
- **Экспорт CSV** — кнопка «Экспорт CSV» в шапке матрицы. Формат: UTF-8 BOM + CRLF (читается в Excel с кириллицей без мусора).

## Риск-скоринг

`score = sum(weight_of_violated) / sum(weight_of_all)`

Веса: `CRITICAL=8`, `HIGH=4`, `MEDIUM=2`, `LOW=1`.

- `score = 0%` → `LOW`;
- `1-30%` → `MEDIUM`;
- `31-70%` → `HIGH`;
- `>70%` → `CRITICAL`.

Уровень отображается цветной плашкой «Риск» в шапке релиза (FR-25).
