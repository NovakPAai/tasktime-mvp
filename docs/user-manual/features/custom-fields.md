# Настраиваемые поля — Руководство пользователя

> Функция: Настраиваемые поля задач
> Доступ: Настройка — ADMIN; Просмотр/заполнение — все
> Обновлено: 2026-03-25

---

## Что такое настраиваемые поля?

Помимо стандартных полей (статус, приоритет, исполнитель), администратор может добавить любые дополнительные поля к задачам:

- Текст (например: «URL задачи в Confluence»)
- Число (например: «Story Points»)
- Флажок (например: «Проверено QA»)
- Дата (например: «Дата демо»)
- Список (например: «Компонент: Бэкенд / Фронтенд / DevOps»)

---

## Просмотр и заполнение полей

На карточке задачи в разделе **Дополнительные поля** отображаются все настроенные поля для этого типа задачи.

Чтобы заполнить поле:
1. Кликните на значение поля
2. Введите или выберите значение
3. Нажмите Enter или кликните вне поля для сохранения

---

## Настройка полей (только ADMIN)

Раздел: **Администрирование** → вкладки «Поля» и «Схемы полей»

### Шаг 1: Создать поле
1. **Администрирование** → **Настраиваемые поля** → **+ Поле**
2. Укажите:
   - Название (например: «Story Points»)
   - Тип (текст / число / флажок / дата / список)
   - Обязательность

### Шаг 2: Создать схему полей
1. **Схемы полей** → **+ Схема**
2. Дайте название (например: «Схема для разработки»)
3. Добавьте нужные поля в схему

### Шаг 3: Привязать схему к типу задачи
1. **Типы задач** → выберите тип (например: TASK)
2. Назначьте схему полей

После этого все задачи типа TASK в проекте с этой конфигурацией будут показывать настроенные поля.

---

## На Kanban-карточке

Если у задачи есть заполненные настраиваемые поля — они отображаются внизу Kanban-карточки (компонент `KanbanCardCustomFields`).

---

## Конфликты схем

Если схема полей изменилась и некоторые задачи имеют значения в полях, которых уже нет в схеме — система покажет предупреждение (модальное окно `SchemaConflictsModal`). Администратор должен разрешить конфликты вручную.
