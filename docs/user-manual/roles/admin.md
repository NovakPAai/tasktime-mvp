# Роль: ADMIN — Руководство

> Обновлено: 2026-03-25

## Что может ADMIN?

Администратор имеет полный доступ ко всем функциям системы кроме назначения роли SUPER_ADMIN.

| Функция | Доступ |
|---------|--------|
| Управление пользователями (создать, изменить роль, заблокировать) | ✓ |
| Создание/удаление проектов | ✓ |
| Управление задачами (все задачи) | ✓ |
| Управление спринтами | ✓ |
| Управление командами | ✓ |
| Просмотр системных метрик и мониторинга | ✓ |
| Просмотр аудит-лога | ✓ |
| Настройка типов задач и схем полей | ✓ |
| Назначение роли ADMIN | — (только SUPER_ADMIN) |

## Типичные сценарии работы

1. **Onboarding нового сотрудника:** Меню → Администрирование → Пользователи → + Пользователь
2. **Блокировка уволившегося:** Найдите пользователя → переключатель «Активен»
3. **Создание проекта:** Меню → Проекты → + Проект
4. **Смена роли пользователя:** Администрирование → Пользователи → Изменить

## Отличие от SUPER_ADMIN

SUPER_ADMIN может назначать роль ADMIN, делать глобальные системные операции (промоут super-admin, ротация паролей). Обычному администратору это недоступно.
