Flow Universe MCP

Руководство по подключению и использованию MCP-сервера для команды разработки и внедрения

🔌Что такое MCP

MCP (Model Context Protocol) — открытый протокол от Anthropic, который позволяет AI-ассистентам (Claude, Cursor, любому MCP-клиенту) получать контекст из внешних систем и выполнять действия в них.

Проще говоря: вместо того чтобы копипастить задачи из браузера в чат с AI — AI сам видит и обновляет Flow Universe напрямую.

🧠
Без MCP
Копируешь задачу из браузера → вставляешь в Claude → Claude отвечает → вручную применяешь изменения
⚡
С MCP
«Взяй TTMP-152, реализуй и закрой» → Claude сам читает задачу, пишет код, логирует время, меняет статус

🏗️Как это работает

┌─────────────────────┐ ┌──────────────────────────┐ │ Claude Code │ │ Flow Universe MCP │ │ Cursor / IDE │────▶│ (HTTP :3002) │ │ Любой MCP-клиент │ │ Bearer Token auth │ └─────────────────────┘ └────────────┬─────────────┘ │ ┌───────────────────────────┤ │ │ ▼ ▼ ┌─────────────────────┐ ┌──────────────────────────┐ │ Backend API │ │ PostgreSQL │ │ (workflow engine) │ │ (прямой доступ) │ │ RBAC + валидация │ │ read + time logs │ └─────────────────────┘ └──────────────────────────┘ ← write ops (статусы, ← read ops + log_time комментарии, задачи) register_ai_session

Ключевые принципы:

  • Операции записи (смена статуса, комментарии, создание задач) идут через Backend API — работает workflow engine и проверяется RBAC
  • Операции чтения и учёт времени — напрямую через Prisma (быстрее, нет бизнес-логики)
  • Агент авторизуется как системный пользователь с ролью MANAGER
  • Каждый запрос к MCP проверяет Bearer Token — без токена запрос отклоняется

⚡Быстрый старт

1
Получи токен доступа

Запроси Bearer-токен у администратора системы (передаётся через защищённый канал — не в открытом чате).

Токены существуют для двух сред: staging (тестирование) и production (боевые данные).

2
Создай файл .mcp.json в корне проекта

Скопируй шаблон из репозитория (.mcp.json в корне) и замени плейсхолдеры на реальные токены.

// .mcp.json
{
  "mcpServers": {
    "flow-universe-staging": {
      "type": "streamable-http",
      "url": "http://5.129.242.171:3002/mcp",
      "headers": {
        "Authorization": "Bearer <STAGING_TOKEN>"
      }
    },
    "flow-universe-prod": {
      "type": "streamable-http",
      "url": "http://72.56.7.218:3002/mcp",
      "headers": {
        "Authorization": "Bearer <PROD_TOKEN>"
      }
    }
  }
}
⚠️

Важно: не коммить .mcp.json с реальными токенами в git. Добавь файл в .gitignore у себя локально.

3
Проверь подключение

В Claude Code (или Cursor) спроси:

get_issue TTMP-1

Если в ответ получаешь карточку задачи — всё работает.

🖥️Настройка в Claude Code и Cursor

Claude Code (CLI)

.mcp.json в корне проекта подхватывается автоматически при запуске claude. Проверить список подключённых серверов: /mcp в интерфейсе.

Claude Desktop

Открыть: Settings → Developer → Edit Config и добавить серверы в claude_desktop_config.json:

{
  "mcpServers": {
    "flow-universe-staging": {
      "type": "streamable-http",
      "url": "http://5.129.242.171:3002/mcp",
      "headers": {
        "Authorization": "Bearer <STAGING_TOKEN>"
      }
    }
  }
}

Cursor

Settings → Cursor Settings → MCP → Add new server → выбрать тип HTTP, указать URL и заголовок Authorization.

MCP Inspector (отладка)

Для проверки инструментов без IDE:

npx @modelcontextprotocol/inspector \
  --url http://5.129.242.171:3002/mcp \
  --header "Authorization: Bearer "

🌐Среды: Dev / Staging / Production

Среда URL Когда использовать Данные
Development stdio (локально) Разработка MCP-инструментов Локальная БД
Staging http://5.129.242.171:3002 Тестирование AI-сценариев Тестовые данные
Production http://72.56.7.218:3002 Работа с реальными задачами Боевые данные ⚠️
🔴

Production — боевые данные. Любое действие через MCP на prod-сервере изменяет реальные задачи, статусы и комментарии. Тестируй сценарии на staging, затем применяй на prod.

📖Инструменты: чтение данных

ИнструментЧто делаетПараметры
get_issue read Карточка задачи: статус, тип, спринт, исполнитель, комментарии, время key: TTMP-95
list_issues read Список задач проекта с фильтрацией projectKey, status, assigneeId, sprintId
get_sprint read Спринт: задачи, статусы, прогресс sprintId
list_sprints read Все спринты проекта projectKey
get_project read Информация о проекте key: TTMP
list_projects read Все проекты системы —
get_time_summary read Разбивка времени Human vs Agent по задаче, стоимость AI-сессий key: TTMP-95

✏️Инструменты: действия и запись

ИнструментЧто делаетПараметры
update_status write Смена статуса задачи через workflow engine (валидирует допустимые переходы) key, status: IN_PROGRESS / REVIEW / DONE / CANCELLED
add_comment write Добавить комментарий к задаче от имени AI-агента key, body
create_subtask write Создать подзадачу (SUBTASK) к существующей задаче parentKey, title, description, priority
claim_issue write Взять задачу в работу: присвоить агенту + перевести в IN_PROGRESS key
complete_issue write Завершить задачу: статус DONE + итоговый комментарий key, summary
fail_issue write Отметить задачу как проваленную агентом: сброс исполнителя + причина key, reason
ℹ️

Workflow engine: update_status проходит через полный цикл валидации. Если переход недопустим по настроенному workflow — вернётся ошибка с объяснением. Это защита от случайных переходов.

⏱️Инструменты: учёт AI-времени

ИнструментЧто делаетПараметры
log_time time Записать время вручную на задачу (source=AGENT) key, hours, note, sessionId
register_ai_session time Записать AI-сессию с метриками токенов и стоимостью. Автоматически раскладывает время по задачам пропорционально issues[] (key+ratio), model, tokensInput, tokensOutput, costUSD

Пример: регистрация AI-сессии

// Агент потратил на 2 задачи: 70% на TTMP-152, 30% на TTMP-153
register_ai_session({
  issues: [
    { key: "TTMP-152", ratio: 0.7 },
    { key: "TTMP-153", ratio: 0.3 }
  ],
  model: "claude-sonnet-4-6",
  tokensInput: 45000,
  tokensOutput: 12000,
  costUSD: 0.87
})

🔧Ошибки и решения

ОшибкаПричинаРешение
401 Unauthorized Неверный или устаревший Bearer-токен Запросить актуальный токен у администратора
Agent login failed 401 Системный агент не может авторизоваться в backend API Администратор проверяет MCP_AGENT_PASSWORD в конфиге контейнера
Issue not found: TTMP-X Задача не существует или проект не тот Проверить ключ задачи в системе. Убедиться, что подключена нужная среда (staging vs prod)
Workflow transition not allowed Переход между статусами запрещён workflow Проверить допустимые переходы в настройках workflow проекта
missing or unknown session ID MCP-клиент не выполнил initialize перед вызовом Нормальное поведение при ручных curl-запросах. В Claude Code/Cursor это делается автоматически
fetch failed / ECONNREFUSED MCP-сервер недоступен Проверить, запущен ли контейнер: docker ps | grep mcp. Сообщить администратору
403 You do not have access to this project Проблема с ролью системного агента Администратор проверяет роль пользователя agent@flow-universe.internal в БД (должна быть MANAGER)

🔒Безопасность

⚠️

Никогда не публикуй Bearer-токены в открытых репозиториях, публичных чатах, скриншотах или документах. Токены дают полный доступ к MCP-операциям.

Что защищает система

  • Bearer Token — каждый запрос проверяется, без токена — 401
  • Workflow engine — write-операции валидируются на допустимость перехода
  • RBAC — агент действует с правами MANAGER, не ADMIN
  • Audit log — все действия агента записываются в audit_log с источником AGENT

Если токен скомпрометирован

Сообщи администратору → токен меняется перезапуском контейнера с новым MCP_SERVICE_TOKEN → старый токен перестаёт работать немедленно.

Ротация токена

# Генерация нового токена
NEW_TOKEN=$(openssl rand -base64 32)

# Перезапуск контейнера с новым токеном (пример для prod)
docker rm -f mcp-flow-universe
docker run -d \
  --name mcp-flow-universe \
  --network tasktime-production_default \
  -p 3002:3002 \
  -e MCP_SERVICE_TOKEN="$NEW_TOKEN" \
  # ... остальные env vars
  ghcr.io/novakpaai/tasktime-backend:<SHA> \
  node dist/mcp/server.js

Создание агента в новой БД

# Если агент ещё не создан
HASH=$(docker run --rm <backend-image> node --input-type=module <<'EOF'
import bcrypt from "bcryptjs";
process.stdout.write(await bcrypt.hash(process.env.PASS, 10));
EOF
)
psql -U tasktime -d <dbname> -c \
"INSERT INTO users (id, email, password_hash, name, role, is_active, created_at, updated_at, is_system)
 VALUES (gen_random_uuid(), 'agent@flow-universe.internal', '$HASH', 'AI Agent',
         'MANAGER', true, NOW(), NOW(), true)
 ON CONFLICT (email) DO UPDATE SET password_hash=EXCLUDED.password_hash, role='MANAGER';"
